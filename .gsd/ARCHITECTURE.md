## Overview

**IntrusionX** is a production-grade DNS threat detection and SIEM (Security Information and Event Management) system. It acts as a DNS proxy that intercepts all DNS queries, analyzes them in real-time using machine learning models and heuristic rules, blocks malicious domains, and visualizes threat activity through a web dashboard.

## Components

### 1. DNS Proxy Server (`dns_server.py`)
- **Purpose:** Intercepts UDP DNS queries, runs threat analysis, proxies clean queries to upstream (8.8.8.8), sinkholes malicious domains
- **Location:** `dns_server.py`
- **Port:** 5054 (non-privileged; configurable to 53 for production)
- **Key features:**
  - Sliding-window DDoS detection (>50 queries/sec per IP)
  - LRU cache (4096 entries) to skip re-analysis of known domains
  - Safe domain lookup table for zero-latency pass-through
  - Logs every query to SQLite

### 2. Threat Engine (`threat_engine.py`)
- **Purpose:** Core ML-powered threat classification engine
- **Location:** `threat_engine.py`
- **Models loaded at startup:**
  - `dga_rf_model.pkl` (~107 MB) вҖ” Random Forest for DGA detection
  - `tunneling_iso_model.pkl` (~1.7 MB) вҖ” Isolation Forest for DNS tunneling
- **Detection layers:**
  - DGA detection (Random Forest probability)
  - DNS tunneling (Isolation Forest anomaly score)
  - DNS Shadowing (>50 unique subdomains for same base domain)
  - Fast Flux (>3 unique IPs resolving same domain)
  - Phishing/Lookalike (Levenshtein вүӨ2 from known brands)
- **Singleton pattern:** `get_engine()` returns cached ThreatEngine instance

### 3. Feature Extractor (`features.py`)
- **Purpose:** Converts raw domain strings into 17-dimensional numerical feature vectors
- **Location:** `features.py`
- **Features extracted:**
  - `length`, `num_subdomains`, `vowel_ratio`, `consonant_ratio`
  - `digit_ratio`, `special_char_count`, `entropy`, `ngram_randomness`
  - `repeated_char_score`, `tld_risk`, `suspicious_keyword_score`
  - `lookalike_score`, `max_consecutive_consonants`, `unique_char_ratio`
  - `hex_char_ratio`, `jaro_winkler_score`, `char_continuity_score`
- **Hard-coded intelligence:** `BRANDS` (32 brands), `PHISH_KEYWORDS` (27 keywords), `HIGH_RISK_TLDS` (20 TLDs)

### 4. Flask API Backend (`app.py`)
- **Purpose:** REST API server serving dashboard data; manages DB, settings, blocklist
- **Location:** `app.py`
- **Key API routes (inferred):**
  - `GET /` вҖ” Dashboard home
  - `GET /api/queries` вҖ” Recent DNS query log
  - `GET /api/alerts` вҖ” Active threat alerts
  - `GET /api/stats` вҖ” Summary statistics
  - `POST /api/blocklist` вҖ” Add domain to blocklist
  - `POST /api/whitelist` вҖ” Add domain to whitelist
  - `GET /api/settings` вҖ” System settings
- **Background threads:** Live query simulator + Feed updater daemon

### 5. Feed Updater (`feed_updater.py`)
- **Purpose:** Daemon thread that periodically fetches external threat intelligence feeds to update the blocklist
- **Location:** `feed_updater.py`
- **Runs every:** 1 hour (configurable)

### 6. SIEM Dashboard (Templates + Static)
- **Purpose:** Real-time web UI for visualizing DNS traffic, threats, and system status
- **Location:** `templates/`, `static/`
- **Pages:** Dashboard, Queries, Threats, Settings
- **Design:** Dark liquid-glass theme, live chart polling

### 7. ML Training Pipeline
- `train.py` вҖ” Basic model training
- `train_models.py` вҖ” Full training with cross-validation (10 KB)
- `enhance_and_retrain.py` вҖ” Dataset augmentation + retraining (29 KB)
- `ingest_datasets.py` вҖ” External dataset ingestion (HuggingFace, CSV)
- `build_database.py` вҖ” SQLite DB population from threat feeds (31 KB)

### 8. Supporting Utilities
- `url_preprocessor.py` вҖ” URL sanitization/normalization
- `feed_updater.py` вҖ” Live threat intelligence feed fetcher
- `attack_simulator.py` вҖ” Simulates DNS attacks for demo/testing
- `benchmark.py` вҖ” Performance benchmarking
- `dummy_data_gen.py` вҖ” Generates synthetic DNS data for training

## Data Flow

1. Client sends DNS query (UDP) вҶ’ `dns_server.py` on port 5054
2. DDoS filter checks query rate per client IP (sliding window)
3. Safe domain LUT checked вҖ” if known-safe, forward immediately
4. LRU cache checked вҖ” if cached result, return immediately
5. `ThreatEngine.analyze(domain, client_ip)` called:
   - `features.py` extracts 17 features from domain string
   - Random Forest predicts DGA probability
   - Isolation Forest scores tunneling anomaly
   - Heuristics check shadowing, fast-flux, lookalike
6. If threat score > threshold вҶ’ sinkhole (return 0.0.0.0), log alert
7. If clean вҶ’ forward to upstream DNS (8.8.8.8), return real IP
8. All queries logged to SQLite `dns_queries` table
9. Flask API reads DB вҶ’ serves dashboard every ~3s via JS polling

## Conventions

- **Naming:** `snake_case` for files, functions, variables. Class names `PascalCase`
- **Structure:** Flat root directory вҖ” all modules co-located, no `src/` hierarchy
- **Error handling:** Bare `except` clauses in several files (e.g., `features.py` fast-flux)
- **Pattern:** Singleton via module-level `get_engine()` function
- **Caching:** `functools.lru_cache` used aggressively for performance
- **Testing:** Ad-hoc `test_*.py` scripts, no pytest framework
